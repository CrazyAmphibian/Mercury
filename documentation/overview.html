Mercury

Mercury is an interpreted dynamically-typed language with a simple but robust syntax.

Why Mercury?

Mercury is here to show that a high-level language doesn't have to compromise on programmer freedom for the sake of simplicity. Drawing from the simplicity of C and ease of use from Lua, Mercury gives the user a lot of control and flexibility, while at the same time, keeping functions atomic and varied.

Mercury is also easy to embed into another program, and has been written in a way that makes it very accessible from the C++ foundation of the language.

Featuring things such as multithreadding, automatic garbage collection, and first class functions, Mercury is more than capable of standing up as a standalone language.

Design philosophy

Mercury is designed around a core set of ideas, listed in order of priority.

Mercury should be:

  • Fast

    • Mercury should be as fast as it can be, without sacrificing other capabilities

  • Simple

    • Mercury's syntax should be easy to read, and not tie multiple behaviors to a single syntax that would cause ambiguity for the user or compiler.

  • Flexible

    • Mercury should be ambivalent to any specific style of programming

    • Mercury should have a diverse set of standard library functions

  • Compatible

    • Mercury should be able to run on any system with the same (or similar) results

    • Mercury should not needlessly remove, deprecate, or change old behavior



Basic Syntax

Hello world

Hello world in Mercury is quite straightforward

print("Hello, world!")

Comments

Comments follow the same syntax as C.

A single line comment is done with two forward slashes

print("Hello, world") //Quite important, do not remove!

A multi-line comment is done as such

/*

this does some stuff.

Pretty cool stuff.

*/

print("Hello, world")

if two comment styles overlap, the multi-line takes priority.

// This does stuff /*

now it doesn't

*/

Order of operations



The program executes in the following order during runtime:

  1. Variable is retrieved

  2. Variable is indexed

  3. Function is called

  4. Unary operations are applied

  5. Binary operations are applied

  6. Variable is set

All operations of the same type are done in left-to-right order, and do not follow standard order of operations.

For example:

b=#a+1/3

first, the variable 'a' is retrieved. Next, the length operator is applied to it. Then, it is added with 1, and divided by 3. Finally, the variable 'b' is set to the result

this example showcases how operations are applied in order

a=5+3/4 //returns 2.0

a=3/4+5 //returns 5.75

a=5+(3/4)//returns 5.75

Strings

Strings are constructed with two quote marks and work like typical strings. Strings will run until they find another quote mark, even if that means spanning multiple lines. Strings accept escapes using backslash, and follow similar rules to C string escapes.

Keywords

Mercury has 17 reserved keywords. These cannot be used for variable names, as they either are part of the language syntax, are represent another variable.

Keywords are case sensitive, so you can use capitalization to share a name with a keyword without any issues.

keywords: Static variables

Static variable keywords represent a constant value, and can be used for assignment or comparison

  • nil

    • represents the nil datatype, nothing.

  • inf

    • represents infinity in as a float.

  • true

    • represents the boolean value of true

  • false

    • represents the boolean value of false

keywords: flow control

These keywords form the structure of a program and are vital for complex behavior

  • end

    • the universal delimiter for statement bodies.

  • if, then, else, elseif

    • forms the basis for if statements

  • while, do

    • forms the basis for while loops

  • continue, break

    • modifies behavior of loops

  • goto

    • unconditional jump to another section of code, use with caution

  • function

    • self-explanitory

  • return

    • can be used in function to abort execution and return values

keywords: scope specifiers

These keywords are used for variable retrieval and assignment to fine tune scope.

  • global

    • specifies that a variable should belong to the top-level state

  • local

    • specifies that a variable should belong to the bottom-most state

Control Flow

If statements

If statements in Mercury operate in a very similar way to those in other high level languages. They are split into 2 parts, the condition and the body. If the condition is true, then the contents of the body are executed. If statements start with if, followed by the condition. Next, the then keyword is used before the body, and is finished with end.

if a>20 then

print("a is kinda big")

end

the else keyword can be used to add more complexity to an if check, and is quite simple to use

if a>20 then

print("a is kinda big")

else

print("a is not that big")

end

the final layer is elseif, which combines else and another if statement together. Elseif can be used as many times as desired, but should go before else.

if a>20 then

print("a is kinda big")

elseif a<0 then

print("a is negative!")

else

print("a is not that big")

end

while loops

As with if statements, while loops are simple in operation

n=0

while n<10 do

do_stuff(n)

n++

end

the continue and break keyword can add further complexity.

n=0

while n<10 do

if should_abort(n) then

break

end

n++

if n%2==0 then

continue

end

stuff(n)

end

functions and returns

Functions are designated with a name and parameters. Do note that a function may receive less parameters than specified.

The return keyword will cause the immediate closure and escape from the current state. Values may be supplied to pass values to the calling state. Returning in the main body of the program will halt all execution.

Functions can be given a name first

function add(num1,num2)

return num1+num2

end

or be assigned to a name

add=function(num1,num2)

return num1+num2

end

using this method allows scope specifiers to be applied to the function for more control.

Goto

Goto can jump code execution to any location within the current state. Caution is advised when using goto, as it can potentially jump over variable declarations and cause bugs with scoping. It's recommended to explicitly state global or local keywords to be safe.

a=0

b=0

while a<100 do

while b<100 do

if something_wrong(a,b) then goto exit end

other_stuff()

end

stuff()

end

exit:

for an example of incorrect usage of goto, this results in an error, since the label thing is not visible within the function state

function stuff()

goto thing

end

thing:

stuff()
